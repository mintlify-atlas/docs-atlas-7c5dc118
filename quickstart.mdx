This guide uses the OpenPanel cloud service at [dashboard.openpanel.dev](https://dashboard.openpanel.dev). If you want to run OpenPanel on your own server, see [Self-hosting overview](/self-hosting/overview) first.

<Steps>
  <Step title="Create an account">
    Go to [dashboard.openpanel.dev](https://dashboard.openpanel.dev) and sign up for a free account.
  </Step>

  <Step title="Create a project">
    After signing in, click **New project** and give it a name. A project groups your applications and websites together.
  </Step>

  <Step title="Get your Client ID">
    Open the project you just created and go to **Settings → Clients**. Copy the **Client ID** — you will need it in the next step.

    <Note>
      The Client ID identifies which project receives your events. It is safe to include in client-side code. Do not confuse it with the Client Secret, which is only required for server-side integrations.
    </Note>
  </Step>

  <Step title="Install the SDK">
    Choose how you want to load the SDK.

    <Tabs>
      <Tab title="Script tag">
        Add the following snippet to the `<head>` of your HTML page. Replace `YOUR_CLIENT_ID` with the value you copied.

        ```html
        <script src="https://openpanel.dev/op1.js" async defer></script>
        <script>
          window.op = window.op || function(...args) {
            (window.op.q = window.op.q || []).push(args);
          };
          window.op('init', {
            clientId: 'YOUR_CLIENT_ID',
            trackScreenViews: true,
            trackOutgoingLinks: true,
            trackAttributes: true,
          });
        </script>
        ```

        <Tip>
          Setting `trackScreenViews: true` automatically records a `screen_view` event on every page navigation, including history API changes in single-page apps.
        </Tip>
      </Tab>
      <Tab title="npm package">
        Install the web SDK with your package manager.

        <CodeGroup>
          ```bash npm
          npm install @openpanel/web
          ```

          ```bash yarn
          yarn add @openpanel/web
          ```

          ```bash pnpm
          pnpm add @openpanel/web
          ```
        </CodeGroup>

        Then initialize the client once at the top level of your app.

        ```typescript
        import { OpenPanel } from '@openpanel/web';

        const op = new OpenPanel({
          clientId: 'YOUR_CLIENT_ID',
          trackScreenViews: true,
          trackOutgoingLinks: true,
          trackAttributes: true,
        });
        ```

        The `OpenPanel` constructor accepts the following options:

        | Option | Type | Description |
        |---|---|---|
        | `clientId` | `string` | **Required.** Your project Client ID. |
        | `clientSecret` | `string` | For server-side use only. Never expose in browser code. |
        | `apiUrl` | `string` | Override the API endpoint. Defaults to `https://api.openpanel.dev`. |
        | `trackScreenViews` | `boolean` | Automatically send `screen_view` on navigation. |
        | `trackOutgoingLinks` | `boolean` | Automatically send `link_out` on external link clicks. |
        | `trackAttributes` | `boolean` | Track clicks on elements with a `data-track` attribute. |
        | `trackHashChanges` | `boolean` | Also fire `screen_view` on hash changes. |
        | `disabled` | `boolean` | Queue all events until `ready()` is called. |
        | `debug` | `boolean` | Log all SDK activity to the browser console. |
      </Tab>
    </Tabs>
  </Step>

  <Step title="Track your first event">
    Call `track()` with an event name and optional properties.

    ```typescript
    // Script tag
    window.op('track', 'signed_up', { plan: 'pro' });

    // npm package
    op.track('signed_up', { plan: 'pro' });
    ```

    Open the **Live** view in your OpenPanel dashboard. You should see the event appear within a few seconds.
  </Step>
</Steps>

## Core SDK methods

### `track(name, properties?)`

Send a custom event. Properties are merged with any global properties you have set.

```typescript
op.track('page_cta_clicked', {
  button: 'Get started',
  location: 'hero',
});
```

### `identify(payload)`

Associate the current session with a known user. Call this after a user signs in.

```typescript
op.identify({
  profileId: 'user_123',       // required
  firstName: 'Jane',
  lastName: 'Smith',
  email: 'jane@example.com',
  properties: {
    plan: 'pro',
    company: 'Acme Corp',
  },
});
```

After `identify()` is called, all subsequent `track()` calls are automatically linked to that user profile.

### `screenView(path?, properties?)`

Manually record a page view. If `trackScreenViews: true` is set, this is called automatically on every navigation.

```typescript
// Use current URL
op.screenView();

// Specify a path explicitly
op.screenView('/pricing', { experiment: 'annual_discount' });
```

### `setGlobalProperties(properties)`

Attach properties to every subsequent event automatically.

```typescript
op.setGlobalProperties({
  app_version: '2.4.1',
  environment: 'production',
});
```

### `clear()`

Reset the profile ID, groups, device ID, and session ID. Call this on sign-out.

```typescript
op.clear();
```
